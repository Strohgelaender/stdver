Standard Versioning

Scheme

M.NmR [.YYYYMMDD] [+HHHH] [-SS]

Examples

1.2.3
1.2.3-GA
1.2b3.20230821-LA
1.2a3.20230821+42FA-XA
1.2a3+42FA-XA

Version Identification Parts

  • M: /\d+/ — Major Version
    Major Version of solution. Bumped on major technical or domain-specific changes only. A bump resets the Minor Version N and Revision R, too. On upgrades human interaction is expected.
  • N: /\d+/ — Minor Version
    Minor Version of the solution within the Major Version. Bumped on new features. A bump resets the Revision R, too. Backward compatibility is seeked for, but not assured. On upgrades human interaction might be required.
  • m: /(a|b|rc|\.)/ — Maturity Level
    Maturity Level within Major Version M and Minor Version N.
    • a: Alpha Maturity Level
      Early version of the solution with incomplete and unstable functionalities to get feedback on product.
    • b: Beta Maturity Level
      Early version of the solution with complete but still unstable functionalities to stabilise product.
    • rc: Release Candidate Maturity Level
      Mature version of solution with complete and stable functionalities to catch last-minute problems.
    • .: Release Maturity Level
      Initial and subsequent versions of the solution with complete and stable functionalities, available for production use.
  • R: /\d+/ — Revision
    Revision of the Maturity Level within Major Version M and Minor Version N. Bumped for every Maturity Level m. On Release Update Maturity Levels, full backward compatibility is assured. On upgrades no human interaction is required.
  • YYYYMMDD: /\d{8}/ — Release Date
    The optional release date in ISO notation.
  • HHHH: /[\dA-F]{4}/ — Source Hash
    The optional hashing checksum of the source code of the solution.
  • SS: /(XA|LA|EA|GA)/ — Availability Scope
    The optional availability scopes of the release.
